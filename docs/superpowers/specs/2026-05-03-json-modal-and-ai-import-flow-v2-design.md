# JSON Paste Modal & AI Import Flow v2 Design

**Date:** 2026-05-03  
**Status:** Approved  

## Goal

Two targeted UX improvements to the SSH host import workflow:

1. **JSON 粘贴导入 Modal** — Replace the cramped inline textarea with a full-screen centered dialog that pre-fills the template JSON as an editable example.
2. **AI 导入对话流 v2** — Update the AI's HOST_IMPORT_PROMPT so step 3 explicitly guides the user to either paste the copied JSON or reply "是" to let the AI import directly.

---

## Change 1: JSON Paste Import — Full-Screen Modal

### Current State

Clicking "JSON" in the ConnectForm sidebar toolbar expands a tiny inline `h-24` textarea directly below the toolbar. It is too small and has no example content.

### Target Behavior

Clicking "JSON" opens a **full-screen overlay modal**:

- **Backdrop:** `fixed inset-0 z-[80] bg-black/60` — clicking backdrop closes the modal and resets state
- **Dialog box:** `fixed z-[81] top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2` — centered, `w-[600px] max-w-[90vw]`, `bg-terminal-surface` rounded border
- **Header:** Title "粘贴 JSON 导入主机" + subtitle "每条记录一台主机，支持 group 子分组（如 Production/Web）" + close ✕ button
- **Textarea:** `h-64` monospace, full-width, resizable vertically — pre-populated with template JSON on first open
- **Pre-population rule:** When modal opens, if `jsonPasteText` is empty, set it to the `TEMPLATE_JSON_EXAMPLE` constant. If it already has content (user typed/pasted something earlier), keep it.
- **Error display:** `text-[11px] text-red-400` inline below textarea on invalid JSON
- **Footer buttons:** "导入到主机列表" (blue, disabled when textarea empty or whitespace-only) + "取消" (muted, resets state)

### Template JSON Example Content

The pre-filled example is identical to the template generated by 下载模板:

```json
[
  {
    "name": "示例服务器",
    "host": "192.168.1.1",
    "port": 22,
    "username": "root",
    "password": "your_password",
    "privateKey": "",
    "group": "Production/Web"
  },
  {
    "name": "开发机",
    "host": "10.0.0.2",
    "port": 22,
    "username": "ubuntu",
    "password": "",
    "privateKey": "-----BEGIN OPENSSH PRIVATE KEY-----\n...",
    "group": "Development"
  }
]
```

### Implementation Scope

**File:** `src/components/ConnectForm.tsx`

- Add `TEMPLATE_JSON_EXAMPLE` constant at module scope (stringified pretty-printed template JSON)
- Remove the inline `{showJsonPaste && <div>...}` expansion below the toolbar
- Add a modal overlay: rendered at the root of the sidebar panel (or at the ConnectForm root), controlled by `showJsonPaste` state
- When `setShowJsonPaste(true)` is called, also call `setJsonPasteText(prev => prev || TEMPLATE_JSON_EXAMPLE)` to pre-fill only when empty
- Closing the modal (backdrop click, cancel button, or ✕) calls: `setShowJsonPaste(false); setJsonPasteText(''); setJsonPasteError(null);`

## Change 2: AI Import Flow v2 — Updated HOST_IMPORT_PROMPT

### Current State

```
HOST_IMPORT_PROMPT:
步骤一：请询问我主机信息（支持多台主机）。
步骤二：根据我的回复，生成一个 JSON 数组，每条记录包含以下字段：name（显示名称）、host（主机地址）、port（端口，默认22）、username（用户名）、password（密码，可选）、privateKey（私钥，可选）、group（分组，可选）。请将 JSON 放在 ```json 代码块中。
步骤三：询问我是否要一键导入到主机列表。
```

Step 3 is vague — the AI just asks a question without guiding the user about the paste option.

### Target Behavior

Updated prompt instructs the AI to:

- **Step 2:** Generate JSON in the exact template format (same field order and format as `TEMPLATE_JSON_EXAMPLE`), including `group` with `/`-separated sub-group example.
- **Step 3:** After outputting the JSON, add a guide message explaining both import options: ①copy the JSON and use the "JSON" paste button in the host list toolbar; ②reply "是" to let the AI import directly.

### Updated HOST_IMPORT_PROMPT

```
请帮我把主机导入到主机列表。
步骤一：请询问我主机信息（支持多台主机）。
步骤二：根据我的回复，生成标准格式的 JSON 数组，放在 ```json 代码块中。每条记录包含：name（显示名称）、host（主机地址）、port（端口，默认22）、username（用户名）、password（密码，可选）、privateKey（私钥路径或内容，可选）、group（分组，可选，支持子分组如 "Production/Web"）。
步骤三：在 JSON 代码块之后，紧接着说：「您有两种方式导入：① 复制以上 JSON，在主机列表底部点击「JSON」按钮粘贴导入；② 直接回复「是」，我来帮您一键导入到主机列表。」
```

### Action Buttons (no change)

The existing "复制 JSON" and "一键导入到主机列表" buttons stay as-is. The prompt update ensures the AI's step 3 message aligns with the button labels already visible to the user.

### Confirm Pattern (no change)

`/^(是|好|确认|yes|一键|import|导入)/i` — unchanged.
